Phase 2 - Detector Build Plan
Status: Ready to build by hand

Important note:
The assignment says AI should not write the detector logic. So the detector code needs to be written by the student. These are the detector pieces that must go into catch.py:

1. Parser
   - Read CSV rows.
   - Parse ts, user, event, result, role, country, city, lat, lon, src_ip.
   - Skip malformed rows without crashing.

2. Impossible travel
   - Only compare successful login events.
   - Compare each user's current successful login with their previous successful login.
   - Use haversine distance.
   - Default threshold: 900 km/h.

3. Off-hours privileged access
   - Flag admin activity outside business hours.
   - Do not flag svc_backup just because it works at night, because it is a service account with a normal off-hours baseline.

4. Brute force
   - Flag many login failures against one user account in a short period.
   - In the data, tkline has repeated failures around 23:11.

5. Password spray
   - Flag one source IP causing failures across many different accounts.
   - In the data, 203.0.113.50 appears across multiple users around 03:31-03:39.

6. Self-scoring
   - Load ground_truth.csv.
   - Compare alert keys against truth keys.
   - Print precision, recall, false positives, and false negatives.
